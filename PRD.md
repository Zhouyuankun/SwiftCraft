# Project Voxel — 产品需求文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在研发一款与 Minecraft 玩法相似的体素游戏 demo，作为技术研究和学习用途。项目非商业化，不涉及侵权问题。通过从零开始使用 C++ 和 Metal 实现体素渲染、相机系统和移动控制，深入理解 3D 游戏引擎核心技术。

### 1.2 项目目标
- 实现一个可运行的 Minecraft-like 体素游戏 demo
- 掌握 Metal 图形 API 在 macOS 平台的应用（通过 metal-cpp 桥接）
- 理解体素渲染、第一人称相机、3D 数学等核心概念
- 建立可扩展的游戏架构，为后续功能扩展奠定基础

### 1.3 核心开发哲学
- **不要过早优化：** 先跑通，再重构
- **保持 C++ 纯粹：** 尽量不混入 Objective-C，利用 `metal-cpp` 进行交互
- **显式控制：** 所有的渲染循环、内存分配都由你自己掌握

### 1.4 功能范围
| 功能 | 状态 |

| 体素（Voxel）生成和渲染 | ✅ 已规划 |

## 3. 功能需求

### 3.1 体素渲染系统
- **体素表示**：立方体作为基础方块单位
- **世界组织**：Chunk 区块（16×16×16），动态加载/卸载
- **方块类型**：草方块、泥土、石头、木头、树叶
- **渲染优化**：面剔除——若一个面的相邻方块也非空气，则不生成该面顶点
- **网格生成**：动态生成顶点和索引缓冲区，按需重建

### 3.2 相机系统
- **视角类型**：第一人称视角（FPS）
- **控制方式**：
  - WASD：前后左右移动
  - 鼠标移动：视角旋转（Pitch/Yaw）
  - 鼠标焦点：点击窗口捕获鼠标，ESC 释放
- **视距设置**：
  - 近裁剪面：0.1 单位
  - 远裁剪面：1000.0 单位
  - 视野角度：65 度
- **移动速度**：可配置（默认 5 单位/秒）

### 3.3 图形渲染
- **渲染 API**：Metal（通过 metal-cpp 桥接）
- **着色器**：Metal Shading Language 编写顶点/片段着色器
- **纹理系统**：
  - 纹理图集包含多种方块纹理
  - 各向同性过滤，支持 Mipmapping
- **渲染效果**：基础纹理映射、透视正确投影、双线性纹理过滤

### 3.4 输入处理
- **键盘输入**：WASD 移动、ESC 退出、Space 上升、Shift 下降
- **鼠标输入**：视角旋转（平滑转动，非离散）、鼠标焦点捕获
- **事件处理**：GLFW 事件轮询，每帧主动查询

### 3.5 地形生成
- **生成算法**：FastNoiseLite（Simplex/Perlin Noise）
- **地形规则**：
  - 根据 (x, z) 坐标通过噪声生成高度 y
  - y=0：石头；y=1~h-2：泥土；y=h-1：草方块
  - 随机分布木头和树叶（生长在草地上）

### 3.6 方块交互
- **射线检测**：从相机中心向前发射射线，判断命中哪个方块
- **左键点击**：破坏命中的方块（设为空气），重建 Chunk 网格
- **右键点击**：在命中面法线方向添加新方块（默认木头）

## 4. 7 个开发里程碑

> 每个里程碑都遵循"最小可行性"原则，确保每一步都能看到即时反馈。

### 里程碑 1：环境搭建与"蓝色窗口"
**目标：** 配置开发环境，确保 C++ 能通过 GLFW 在 Mac 上打开一个 Metal 驱动的窗口。

**技术细节：**
1. 集成 `glfw` 库处理窗口和输入
2. 集成 `metal-cpp` 桥接 C++ 和 Metal API
3. 初始化 `MTL::Device` 和 `MTL::CommandQueue`
4. 通过 `MTL::RenderPassDescriptor` 的 `clearColor` 设置纯蓝色背景

**验证点：** 屏幕显示纯蓝色窗口，渲染循环正常运行。

---

### 里程碑 2：渲染第一个三角形（2D 基础）
**目标：** 跑通 Metal 的图形管线（Pipeline）。

**技术细节：**
1. 编写最简 `.metal` Shader（顶点和片元）
2. 创建 `MTL::RenderPipelineState`
3. 使用 `MTL::Buffer` 存储顶点坐标（CPU 传输到 GPU）
4. 在渲染循环中发起绘制调用

**验证点：** 窗口中央出现一个固定的彩色三角形。

---

### 里程碑 3：3D 变换与第一个立方体
**目标：** 引入数学库，让画面从 2D 变成 3D。

**技术细节：**
1. 引入 **GLM** 数学库（处理矩阵）
2. 实现 **MVP 矩阵**（Model, View, Projection）
3. 在 Shader 中添加 `constant` 缓冲区来接收矩阵
4. 将三角形的两个顶点扩展成立方体的多个三角形

**验证点：** 看到一个具有透视感的立方体（Voxel）。

---

### 里程碑 4：摄像机系统与输入监听
**目标：** 让你可以像玩游戏一样在空间中自由移动。

**技术细节：**
1. 监听 GLFW 的键盘（WASD）和鼠标（平滑转动）事件
2. 根据鼠标位移更新摄像机的 `Pitch` 和 `Yaw`
3. 每帧更新 View 矩阵并传给 Metal
4. 实现鼠标焦点捕获/释放（点击窗口捕获，ESC 释放）

**验证点：** 你可以绕着那个立方体上下左右飞行观察。

---

### 里程碑 5：体素区块（Chunk）与面剔除
**目标：** 解决性能问题，从"渲染一个方块"进化到"渲染一组方块"。

**技术细节：**
1. 定义 `Chunk` 类（16×16×16 的 3D 数组）
2. **核心优化（面剔除）：** 遍历方块，如果一个面被邻居遮挡，则不生成该面的顶点数据
3. 将 Chunk 数据整合进一个大的顶点缓冲区
4. 实现 Chunk 的网格重建机制

**验证点：** 渲染出一个由数百个方块组成的实心大方块，且帧率稳定在 60fps。

---

### 里程碑 6：地形生成（噪声算法）
**目标：** 让世界不再是平坦的，而是有起伏的山脉。

**技术细节：**
1. 集成 **FastNoiseLite**（或手写 Perlin Noise）
2. 根据 (x, z) 坐标生成高度 y
3. 根据高度填充方块类型（草地、泥土、石头）
4. 实现 Chunk 间的噪声连续性

**验证点：** 进入游戏后看到一片随机生成的起伏山丘。

---

### 里程碑 7：交互——破坏与放置
**目标：** 实现游戏的核心玩法逻辑。

**技术细节：**
1. **射线检测（Raycasting）：** 从相机中心发射射线，判断命中了哪个方块
2. **左键点击破坏：** 修改数组数据，重新构建（Re-mesh）该 Chunk 的顶点缓冲区
3. **右键点击放置：** 在命中面的法线方向添加一个新方块
4. 处理跨 Chunk 边界的情况

**验证点：** 你可以挖坑或者盖房子了。

## 5. 非功能需求

### 5.1 性能要求
- **帧率**：稳定 60fps（在支持 60Hz 刷新率的显示器上）
- **内存使用**：峰值内存占用不超过 500MB
- **启动时间**：冷启动时间小于 5 秒
- **渲染延迟**：输入到渲染延迟小于 50ms

### 5.2 兼容性要求
- **操作系统**：macOS（需支持 Metal 的 GPU）
- **硬件要求**：
  - 支持 Metal 的 Mac 设备
  - 至少 4GB 可用内存
- **分辨率支持**：支持从 800×600 到 4K 的各种分辨率

### 5.3 可维护性
- **代码结构**：模块化设计，清晰的责任分离
- **文档要求**：关键算法和类需要文档注释
- **配置管理**：图形设置和输入映射可外部配置

- **Texture Atlas**：包含多个纹理的单一纹理图像
- **Raycasting**：射线检测，用于方块选择
